     - Pobierz profil: `getBusinessProfile()` (z onboarding-dal.ts)

     - Sprawdz cache: `getSuggestionWithCacheCheck(chapterId, inputHash, profile?.profile_version ?? 0)`

     - `buildSuggestionPrompt({chapterTitle, content, courseTopic, profile})`
     - `generateObject({ model: getModel('suggestions'), schema: suggestionOutputSchema, ...})`

  6. Save:
     - `saveSuggestion({...})` z danymi z AI + user_id, course_id, chapter_id, model_name, input_hash, profile_version
  7. Return: `{ suggestion, remaining }` z 200

<task type="auto">
  <name>Task 2: Hook useChapterSuggestion</name>
  <files>src/hooks/use-chapter-suggestion.ts</files>
  <action>
  Stworz `src/hooks/use-chapter-suggestion.ts` — wzorzec z `use-chapter-images.ts` (uproszczony, sync JSON):

  ```typescript
  'use client'
  ```

  Hook przyjmuje:
  - `chapterId: string`
  - `initialSuggestion?: BusinessSuggestion | null` — z server component (persystencja po refresh)

  Stan:
  - `suggestion: BusinessSuggestion | null` — inicjalizowany z `initialSuggestion`
  - `isGenerating: boolean` — spinner/loading state
  - `remaining: number | null` — ile generowań zostalo (null = nie wiadomo jeszcze)
  - `error: string | null` — komunikat bledu

  Callbacki (useCallback):
  - `generate(params: { courseId, content, chapterTitle, courseTopic?, force? })`:
    * Guard: jesli `isGenerating` → return (double-click guard)
    * setIsGenerating(true), setError(null)
    * fetch POST `/api/business-ideas/generate` z { chapterId, ...params }
    * 429 → setRemaining(0), toast.error('Dzisiejszy limit zostal wykorzystany')
    * !ok → setError('Nie udalo sie wygenerowac sugestii'), toast.error(...)
    * ok → setSuggestion(data.suggestion), setRemaining(data.remaining)
    * finally: setIsGenerating(false)

  - `bookmark()`:
    * Jesli brak suggestion → return
    * Optimistic update: setSuggestion({...suggestion, is_bookmarked: !suggestion.is_bookmarked})
    * Server action: `bookmarkSuggestion(suggestion.id)`
    * Toast: "Pomysl zapisany" lub "Pomysl usuniety z zapisanych"
    * Jesli blad → rollback optimistic update

  - `dismiss()`:
    * Jesli brak suggestion → return
    * Optimistic update: setSuggestion(null)
    * Server action: `dismissSuggestion(suggestion.id)`
    * Toast: "Pomysl ukryty"
    * Jesli blad → rollback

  - `refresh(params)`:
    * Wywolaj `generate({...params, force: true})`

  Deps useCallback: uzywaj tylko stabilnych referencji. Nie dodawaj `suggestion` do deps `generate` — uzyj ref jesli potrzeba.

  Return: `{ suggestion, isGenerating, remaining, error, generate, bookmark, dismiss, refresh }`
  </action>

  - Double-click guard na generate
